# Learning Coach Question Types

Use these question patterns to check understanding without creating full coding exercises.

## 1. Purpose Questions

Use when checking whether the user understands why a concept exists.

Examples:

- What problem does this concept solve?
- What becomes annoying without this concept?
- Why did programmers invent this?
- What would code look like if this concept did not exist?

## 2. Machine / System Questions

Use when checking the underlying mechanism.

Examples:

- What is the computer actually doing here?
- What is stored?
- What changes?
- What waits?
- What gets called?

## 3. State + Transformation Questions

Use when the concept changes something over time.

Examples:

- What state exists before this line?
- What transformation happens?
- What is the new state afterward?
- Which part of the code caused the change?

## 4. Information Flow Questions

Use for functions, APIs, promises, events, data structures, and modules.

Examples:

- What goes in?
- What transformation happens?
- What comes out?
- Where does this value travel next?
- What boundary does this information cross?

## 5. Pattern Recognition Questions

Use to compress new ideas into old patterns.

Examples:

- What old thing is this really?
- Is this storage, transformation, control flow, organization, or boundary?
- What larger pattern does this belong to?
- Where else have you seen this same idea?

## 6. Syntax Mapping Questions

Use after the mental model is mostly clear.

Examples:

- Which part of the syntax represents the input?
- Which part represents the transformation?
- Which part represents the output?
- What is language-specific here, and what is universal?

## 7. Beginner Trap Questions

Use to detect fake intuition.

Examples:

- What explanation sounds intuitive but breaks later?
- What do beginners usually memorize here?
- Where does the simple analogy fail?
- What would be a wrong but tempting answer?

## 8. Transfer Questions

Use to test cross-language understanding.

Examples:

- How would the same idea appear in Python?
- What changes in syntax but not in concept?
- Is this still the same pattern in another language?
- What part is universal?

## 9. Tiny Code-Reading Questions

Use a very small snippet, but do not make it a full exercise.

Example:

```ts
const result = double(4)
```

Question:

- What information flows into `double`, and what should flow out?

## 10. Production Bridge Questions

Use when the user wants real-world thinking.

Examples:

- Where might this appear in a real app?
- What scale problem does this solve?
- When would this be overkill?
- What would an enterprise team care about here?
