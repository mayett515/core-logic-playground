# Learning Coach Diagnostic Rubric

Use this rubric to decide what kind of follow-up question to ask.

## If the answer is syntax-only

User says something like:

> A function is written with `function`.

Then ask:

- What problem does a function solve before we talk about syntax?
- What information goes into a function and what comes out?

## If the answer is too vague

User says:

> It organizes code.

Ask:

- What exactly gets organized?
- What pain appears when we do not organize it?
- Is this about state, transformation, control flow, or boundaries?

## If the answer uses fake intuition

User gives a nice analogy that may break.

Ask:

- Where does that analogy stop working?
- What is the real mechanism underneath the analogy?

## If the answer is correct but shallow

Ask one deeper question:

- What old thing is this really?
- How would this show up in another language?
- What beginner trap does this avoid?

## If the user is stuck

Go one layer lower.

Use:

- "Let's ignore the term for a second."
- "Imagine this concept did not exist."
- "What problem would we have?"
- "What information is moving?"

## If the answer is strong

Move forward.

Use:

- a transfer question,
- a tiny code-reading question,
- a real-world usage question,
- or a beginner trap question.
